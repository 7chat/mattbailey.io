CSS preprocessors rock, fact!

As a long time LESS user I don’t need any convincing about the benefits of CSS pre-processors. If you’re not using one already you really should be - my productivity has increased exponentially as a result. I’ve used LESS on all of GPMD’s projects since I started working here; The Watch Gallery, Barnard Levit and Budd Shirts to name a few.

LESS or SASS?

LESS has played a major part in our front-end design process through the use of fantastic frameworks and tools such as Bootstrap and The Semantic Grid System. I’ve been aware of SASS for about as long as LESS (maybe even longer) but for a number of reasons I’ve never gone down that path. The main reason for me is that LESS seemed a bit more approachable - It has a nice, easy-to-follow website, and it doesn’t require any arcane knowledge of ruby gems (say what!?) or command-line-fu.

Command line fears

However, even though I’m a designer first and foremost, as time has gone by I’ve started using the command line more and more. Git is now a staple part of my daily diet, and I’ve started looking more into front-end process automation (see my article on Grunt for beginners). My plan is to build on the front-end process here at GPMD, introducing tools such as Yeoman - comprising of Yo for scaffolding, Bower for dependency management, and Grunt for the build process.

As a result of all this other command line ‘stuff’ I’m exposing myself to, I’m finding myself not so scared of the command line any more, and honestly, installing and using SASS really isn’t that tricky. In fact, if you don’t want to touch the command line you don’t have to - just download CodeKit (if you’re on a Mac - Not sure if there’s an equivalent Win/Linux app?), write your SASS, and let CodeKit do all the hard work!

But even so, why change from LESS to SASS?

Firstly, let me just set the record straight - LESS works for me. It does everything I want it to do, and does it well. If you’re already using LESS and are happy with it then don’t feel pressured to change - not only is it brilliant, but it’s in active development so will be around for the foreseeable future! But just because something works doesn’t mean there isn’t something better you could use - I don’t agree with the old adage “if it ain’t broke, don’t fix it” - that’s totally counter intuitive to progress people!

I’m also not ashamed to admit that I’m easily influenced. In order to stay in touch with the web design world at large I listen to quite a few design/front-end podcasts. Overwhelmingly most of the designers/front-end devs that I admire and respect (I’m talking the Chris Coyiers of this world) are using SASS. I want to see what all the fuss is about, and I don’t want to be left behind - that’s just me.

I’m also aware that SASS is more ‘powerful’ than LESS, with added coolness such as nesting media queries, proper logic and looping capabilities, the ability to @extend styles and so on. Although this hasn’t been an issue for me yet, I can see a time when some of these things will probably prove to be incredibly useful.*

*Update, 3 Aug 2013: As @Gorka points out in the comments below, as of v1.4.0 LESS now supports nested media queries, looping (of sorts), extends and so on.

Gettin’ sassy with SASS

So the long and short of it is this: SASS is something I’ve been flirting with for ages - a little nudge here and a little wink there - and I’ve finally drummed up the courage to ask her out on a date! I’m a little apprehensive, but excited, and I have a good feeling about this relationship. Wish me luck, and I’ll let you know how it goes!
